## 🗂️ 目录说明

### agents/
包含所有 AI Agent 的实现，每个 agent 负责工作流的一个阶段。

### skills/
技能模块，包装 agents 提供更高级的功能：
- `enhanced_code_skill.py`: 基础的 6 阶段工作流
- `self_healing_skill.py`: 带自愈功能的增强版

### mcp_servers/
MCP (Model Context Protocol) 服务器，提供文件系统和 Git 操作能力。

### config/
Agent 配置文件，包含模型参数、提示词等。

### tests/
所有测试文件，包括单元测试、集成测试、UI 测试等。

### debug/
调试脚本，用于开发过程中的问题排查。

### examples/
示例代码，展示如何使用各个功能。

### docs/
英文文档目录，保持与国际标准兼容。

### 文档/
中文文档目录，分为：
- **开发文档/**: 开发相关的文档
- **历史文档/**: 项目演进过程中的历史文档

### 按功能查找

**需要了解项目？**

**需要使用自愈功能？**
→ `自愈功能快速开始.md` 或 `docs/SELF_HEALING_SKILL.md`

**需要开发文档？**
→ `文档/开发文档/`

**需要运行测试？**
→ `tests/`

### 按类型查找

**Markdown 文档**: 根目录 + `文档/` + `docs/`
**Python 代码**: `agents/`, `skills/`, `mcp_servers/`
**测试文件**: `tests/`
**配置文件**: `config/`, `.env`
**示例代码**: `examples/`

## 📝 文件命名规范

### Python 文件
- 模块文件: `snake_case.py`
- 测试文件: `test_*.py`
- 调试文件: `debug_*.py`

### 文档文件
- 中文文档: 使用中文命名，如 `项目概览.md`
- 英文文档: 使用英文命名，如 `README.md`
- 保持简洁明了，避免过长的文件名

### 目录命名
- 英文目录: 小写，如 `agents/`, `tests/`
- 中文目录: 使用中文，如 `文档/`, `开发文档/`
